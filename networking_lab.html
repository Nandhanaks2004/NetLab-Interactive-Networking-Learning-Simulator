⚡ Encapsulation Flow

Each layer wraps data with its own header before passing it down.

[ HTTP DATA ] ← Application (L7)
[ TCP Header | HTTP DATA ] ← Transport (L4)
[ IP Header | TCP Header | HTTP DATA ] ← Network (L3)

[ 10110011 01001101 ... ] ← Physical (L1) BITS

⚠️ SYN Flood Attack
🔴 Attack Type: DoS / DDoS

An attacker sends thousands of SYN packets with spoofed source IPs. The server responds with SYN-ACK but never receives the final ACK.

Result: Server's connection table fills up → legitimate users can't connect.

Defense: SYN cookies, rate limiting, firewall rules, IDS/IPS systems.

🌍 IPv4 vs IPv6

IPv4

  • 32 bits → ~4.3 billion addresses
  • Format: 192.168.10.5
  • Running out! Solved by NAT + CIDR
  • Still most common today

IPv6

  • 128 bits → 340 undecillion addresses
  • Format: 2001:0db8:85a3::8a2e:0370:7334
  • Enough for every grain of sand on Earth
  • Built-in security (IPsec)

🧠 How Traceroute Works (TTL Trick):

Each packet gets a TTL (Time-To-Live) counter. It starts at 1, then 2, then 3...
When TTL hits 0 at any router, that router sends back "TTL exceeded" — revealing its IP.
This maps the full path hop by hop!

🧠 What is Subnetting?

Subnetting is the process of dividing one large network into smaller sub-networks. Like dividing a city into neighborhoods.

Before Subnetting:
192.168.1.0/24 → 254 hosts, 1 big network

After Subnetting into 4:
192.168.1.0/26 → 4 × 62 hosts

Key Formulas

2ⁿ = number of subnets (n = bits borrowed)
2ʰ − 2 = usable hosts (h = host bits, -2 for network & broadcast)
256 − subnet mask octet = block size

🧮 Subnetting Step-by-Step: 192.168.1.0/24 → 4 Subnets

Click each step to reveal the solution process:

1
Determine Bits to Borrow

We need 4 subnets. Use the formula 2ⁿ = 4 → n = 2

So we need to borrow 2 bits from the host portion.

2¹ = 2 subnets → need 2? No.
2² = 4 subnets → YES! Borrow 2 bits.
2
Calculate New Prefix

Original prefix: /24 (24 network bits)

Borrowed bits: 2

New prefix: /24 + 2 = /26

CategoryBeforeAfter
Network Bits2426
Host Bits86
Subnet Mask255.255.255.0255.255.255.192
3
Calculate Hosts Per Subnet

Host bits remaining: 6

Formula: 2ʰ − 2 = 2⁶ − 2 = 64 − 2 = 62 usable hosts

Why subtract 2?
1 address reserved for Network ID (first address)
1 address reserved for Broadcast (last address)
4
Find Block Size

The block size tells us the increment between subnets.

In /26, the last octet of subnet mask is 192.

Block size = 256 − 192 = 64

Or: The last borrowed bit's positional value in an octet (128, 64, 32...) → 2nd bit = 64

Bit values: 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1
We borrowed bits 128 and 64
Last borrowed bit = 64 → Block Size = 64
5
List All Subnet Ranges

Start at 0, increment by 64 until reaching 256:

Subnet 1192.168.1.0/26
NET: .0 | HOST: .1-.62 | BC: .63
Subnet 2192.168.1.64/26
NET: .64 | HOST: .65-.126 | BC: .127
Subnet 3192.168.1.128/26
NET: .128 | HOST: .129-.190 | BC: .191
Subnet 4192.168.1.192/26
NET: .192 | HOST: .193-.254 | BC: .255

⚠️ Common Subnetting Mistakes
❌ Forgetting to subtract 2 from hosts
Wrong: 2⁶ = 64 hosts. Correct: 2⁶ − 2 = 62 usable hosts. Always subtract Network ID and Broadcast.
❌ Wrong bit borrowing calculation
Need 5 subnets? 2² = 4 (not enough!). Must use 2³ = 8. Always round UP to next power of 2.
❌ Confusing block size with number of hosts
Block size = 64 means subnets start at 0, 64, 128, 192. Usable hosts = 62 (not 64!).
❌ Not checking if there are enough host bits left
In /30, you have only 2 host bits → 2² − 2 = 2 usable hosts. Check h > 0 before subnetting!
🔐 Security Benefits of Subnetting
🧱
Network Segmentation
Divides the network into isolated zones. HR, Finance, and Guest on separate subnets — they can't talk without a router's permission.
🛑
Limits Attack Spread
If malware infects one subnet, it's contained. Without subnetting, a single worm can spread across all 254 hosts instantly.
🎯
Better Firewall Control
Firewall rules can be applied per-subnet. Block all traffic from Guest VLAN to Finance subnet with a single rule.
📊
Reduced Broadcast Traffic
Smaller subnets = smaller broadcast domains = less network noise = better performance and easier monitoring.

  • Creates ICMP Echo Request
  • DNS lookup for destination
  • TCP/IP stack builds packet
  • ARP to find gateway MAC

  • Reads destination MAC
  • Forwards to correct port
  • No IP processing
  • Builds MAC table

  • Reads destination IP
  • Checks routing table
  • NAT: replaces private IP
  • Decrements TTL

  • Multiple hops
  • Each checks routing table
  • TTL decremented each hop
