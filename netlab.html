The Postal Analogy 📬

Think of a network like a mail system:
▸ Your device = You writing a letter
▸ IP address = House address on the envelope
▸ Router = Post office sorting letters
▸ Switch = Postman on your street
▸ DNS = Directory to find addresses
▸ Protocol = Rules for how mail is handled

// 10 — CIDR & Routing
Classful vs Classless addressing, and how routing works
Classful Addressing (Old Way)
Class A — 1.x.x.x to 126.x.x.x
16M hosts, only 128 networks. Huge waste!
Class B — 128.x.x.x to 191.x.x.x
65K hosts per network
Class C — 192.x.x.x to 223.x.x.x
254 hosts, many networks
⚠️ Problem: A company needing 500 hosts must take a Class B (65K addresses) — wasting 64,000+ IPs!
CIDR — Classless Inter-Domain Routing
✅ CIDR lets you define ANY prefix length, eliminating waste and enabling flexible allocation.
Need 500 hosts?
Use /23 → 512 addresses, 510 usable

Need 1000 hosts?
Use /22 → 1024 addresses, 1022 usable

Need exactly 30 hosts?
Use /27 → 32 addresses, 30 usable ✓
CIDR notation: IP/prefix-length e.g. 192.168.1.0/24
Routing — How Packets Find Their Way
Static Routing
An admin manually enters routes.
✅ Simple, predictable, no CPU overhead
❌ Doesn't adapt to failures
Used in: Small networks, last-resort routes
ip route 10.0.0.0 255.0.0.0 192.168.1.1
Dynamic Routing
Routers share routes automatically.
✅ Adapts to network changes
❌ More complex, uses bandwidth
Protocols: OSPF, BGP, RIP, EIGRP
router ospf 1
network 10.0.0.0 area 0
